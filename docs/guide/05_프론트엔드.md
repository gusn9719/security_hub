| 기능 | 구현 | 권한 |
|---|---|---|

| INP-03 클립보드 감지 | `Clipboard.getData()`, Android 12+ 시스템 알림(~1.8초) 뒤에 자체 배너 표시, 7초 자동 해제. **OverlayEntry 안 쓰고 setState 기반 in-tree 위젯**(프레임워크 quirk 회피, 의도적 선택) | 없음 |

## 7-A WebView 통합의 디테일

`sandbox_browse_screen.dart`가 JS 주입으로 처리하는 것들:
- **WebSocket 재작성**: `window.WebSocket`을 `Proxy`로 가로채 `wss://임의호스트`
  를 `ws://서버/sandbox/browse/{id}/novnc`로 재작성(자체서명 인증서 문제 회피).
- **절대경로 fetch/XHR 재작성**: `/api/...` 같은 절대경로를
  `pathBase + 그경로`로 변환.
- **세션 종료 감지**: 8초 폴링(`#noVNC_status` 텍스트에 "disconnect" 포함
  검사) + `MutationObserver`(동적 추가 노드에 "session"+"expired/timeout"
  텍스트 검사) 2중 감지.
- **터치 오버레이**: noVNC 기본 터치 핸들러가 Pointer Lock으로 캔버스를
  독점하는 문제를 막기 위해 투명 `<div>`를 캔버스 위에 깔고 탭/스와이프를
  자체 판정(10px 이하 이동=탭→마우스클릭, 그 이상=스와이프→WheelEvent).
- **한글 IME**: `compositionend` 이벤트를 noVNC의 키보드 입력 핸들러용
  `InputEvent`로 변환 중계.
- **투표 모달 await 수정(P0-6)**: 예전엔 `submitVote()`를 fire-and-forget
  으로 호출해서 네트워크 오류 시 사용자는 "저장됨"으로 봤지만 DB엔 없었음.
  지금은 `await`로 결과를 확인하고, 실패 시 `_voteDone`을 갱신하지 않아
  다음 종료 시 재시도 가능하게 함.
- **위협 자동 차단(DC-34) 연동**: 시작 직후부터 2초 간격 `_pollStatus()`로
  `/sandbox/browse/{id}/status`를 조회, `threat_detected=true`면
  `_buildThreatOverlay()`(스크린샷 + 차단 안내 + "메인으로 돌아가기" 버튼,
  투표 모달을 거친 후 이동).
