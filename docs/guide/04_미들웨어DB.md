# 4. 보안 미들웨어 체인 / DB 스키마

> 전체 통합본: [../상세 파악 가이드.md](../상세%20파악%20가이드.md) (섹션 6~7)
> 인덱스: [00_인덱스.md](00_인덱스.md)

## 보안 미들웨어 체인

`backend/main.py`. 실행 순서(바깥→안쪽): **CORS → SecurityHeaders →
RateLimit → DeviceUUID → OptionalAuth → BlockDangerousMethods → 핸들러**

| 미들웨어 | 역할 | 핵심 |
|---|---|---|
| CORS | `allow_origins=["*"]` | TRACE/CONNECT/TRACK은 `allow_methods`에서 명시적으로 제외 |
| SecurityHeaders | `X-Content-Type-Options: nosniff`, `X-XSS-Protection`, `Referrer-Policy: no-referrer`, `Cache-Control: no-store`(NF-12), CSP | noVNC 정적자산(`.js/.css/.png` 등)만 캐시 허용. `/test-phishing`은 `unsafe-inline`+`blob:` CSP(테스트 페이지 특성상 필요) |
| RateLimit | IP 기반(NF-24) | `/analyze` 10/min, `/sandbox/browse`·`/auto-test` 5/min, `/sandbox/votes` 20/min, `/auth/kakao` 5/min. IP 추출: `cf-connecting-ip`→`x-real-ip`→`x-forwarded-for`→`client.host` 순 |
| DeviceUUID | `X-Device-UUID` 강제(NF-30) | UUID v4 검증, 없으면 401, 형식오류면 400. 제외경로: `/docs` 등 + 정규식 `^/sandbox/browse/[^/]+/novnc(?:/|$)`(DC-43 — 이전엔 `"/novnc" in path` 단순 부분일치라 `/api/novnc-test` 같은 경로로 우회 가능했음) |
| OptionalAuth | JWT 선택적 검증 | [03_투표인증.md](03_투표인증.md) 참조 |
| BlockDangerousMethods | TRACE/CONNECT/TRACK → 405 | XST 공격, 프록시 터널링 악용 차단 |

**전역 예외 핸들러**: 클라이언트에는 일반 메시지만(`서버 내부 오류가
발생했습니다`), 스택 트레이스는 서버 로그에만.

**RateLimit 메모리 누수 수정(DC-44)**: 윈도우 밖 타임스탬프만 정리하고
빈 리스트 키를 `dict`에 남겨두면 IP×엔드포인트 조합 수만큼 키가 영구
누적된다 — `.get(key, ())`로 읽어서 비어있으면 키 자체를 안 만들도록 수정.

**미해결 한계 (P1-4)**: `x-forwarded-for`를 무조건 신뢰 — Cloudflare Tunnel
없는 환경에서는 클라이언트가 이 헤더를 위조해 레이트리밋을 우회할 수 있음
(CF Tunnel 운영 시에만 `cf-connecting-ip`가 신뢰할 수 있는 값).

## DB 스키마

SQLite `backend/security_hub.db`, WAL 모드 + `busy_timeout=5000` +
`synchronous=NORMAL`. **읽기/쓰기 커넥션 분리(DC-17)**: 런타임 요청은
`get_ro_connection()`(`mode=ro` URI — 의도하지 않은 쓰기 시 즉시
`OperationalError`), 스크립트/초기화만 `get_rw_connection()`.

| 테이블 | 핵심 키/제약 | 역할 |
|---|---|---|
| `blacklist` | `url_hash` UNIQUE, `domain`/`registered_domain` 인덱스 | C-TAS 피싱 URL |
| `whitelist` | `domain` UNIQUE, `match_mode`(exact/suffix) | 안전 도메인. `pattern` 모드는 DC-001로 제거 |
| `domain_reputation_cache` | `registered_domain` UNIQUE, `expires_at` | WHOIS/SSL 캐시 TTL 7일 |
| `url_votes` | `session_id` UNIQUE, 부분 `UNIQUE(device_uuid, registered_domain) WHERE vote IN (...)`, `user_id NULL`(AUTH-01) | 7-A 투표. `CHECK(vote IN ('safe','danger','spam','unsure'))` |
| `sandbox_results` | `session_id` PK(DC-33로 `url_hash UNIQUE` 제거) | 7-A/7-B 결과, TTL 24h |
| `analysis_history` | `url_hash`, `analyzed_at` 인덱스 | 분석 이력(BackgroundTasks 비동기 INSERT, DAT-06) |
| `users` | `kakao_id` UNIQUE(AUTH-01) | 카카오 가입자. FK 미설정(SQLite `foreign_keys=OFF`, 탈퇴 흐름은 범위 밖) |

**마이그레이션 전략**: `PRAGMA table_info`로 컬럼 존재를 확인하고 없으면
조건부 `ALTER TABLE`로 추가 — 기존 DB를 깨지 않고 스키마를 진화시킴
(예: `whitelist.match_mode/risk_level/note/registered_domain`,
`url_votes.user_id`).
