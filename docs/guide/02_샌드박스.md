## 7-A 직접 탐방

모듈: `backend/services/browse_service.py` + `backend/routers/sandbox.py`

| 항목 | 값 |
|---|---|
| Docker 이미지 | `kasmweb/chromium:1.14.0` |
| 포트 | 6901(HTTPS, KasmVNC) |
| 동시 한계 | 4세션(`asyncio.Semaphore(4)`), RAM 512MB×4 |
| Pre-warm pool | 2개 컨테이너 상시 대기 |

**아키텍처**: `Flutter WebView → (HTTP 평문) → FastAPI SSL-strip 프록시
(127.0.0.1:동적포트) → KasmVNC HTTPS(자체서명 인증서)`. Android WebView가
자체서명 wss:// 인증서를 신뢰하지 않는 문제를 백엔드 프록시로 우회.

**Pre-warm pool**: 서버 부팅 시 컨테이너 2개를 미리 만들어두고, 요청이 오면
`_pool_idle` 큐에서 즉시 할당(pool hit, 응답 <5초) + 백그라운드로 보충.
풀이 비었으면 온디맨드 생성(pool miss, 30~80초 — 콜드스타트는 어쩔 수 없음).

**SSL-strip 프록시** (`_proxy_handle`): HTTP 헤더 파싱 → Authorization
Basic 헤더 자동 주입(VNC_PW는 컨테이너마다 `uuid4().hex`로 랜덤, DC-27) →
타겟에 TLS로 재연결 → WebSocket이면 바이트 양방향 릴레이, HTML이면 절대경로
재작성(`src="/"` → `/novnc/`) + CSP 메타태그 제거 + `fetch`/`XHR` 패치
스크립트 주입(상대경로를 `/sandbox/browse/{id}/novnc/...`로 재작성).

**noVNC WebSocket 프록시** (`routers/sandbox.py`의 `_novnc_ws_proxy`):
`websockets.connect(wss://127.0.0.1:{kasm_port}/, ...)`로 직접 연결,
`ping_interval=30`(Cloudflare Tunnel 100초 타임아웃 방어), `max_size=10MB`.

**CDP 위협 자동 차단 (DC-34)** — ⚠️ **CLAUDE.md 구버전 기록은 "미구현"이라고
적혀 있었으나 실제로는 백엔드+프론트엔드 양쪽 다 완전히 구현돼 있음을
2026-06-20에 코드로 직접 확인하고 문서를 정정함** (`CLAUDE.md` 수정 완료). 동작:
1. 컨테이너 내부에서 Python CDP 모니터 스크립트를 `exec_run(stream=True,
   tty=True)`로 실행 — `tty=True`가 핵심(버퍼링 우회, 이벤트 즉시 전달).
2. `Page.frameNavigated` 이벤트마다 `check_blacklist()` 재매칭. 히트하면
   스크린샷(`Page.captureScreenshot`) 캡처 → `_threat_cache[container_id]`
   기록 → `terminate_browse_session()` 즉시 종료.
3. `Page.downloadWillBegin` 이벤트는 무조건 즉시 종료(악성 파일 다운로드
   시도 자체가 위협 신호).
4. `GET /sandbox/browse/{id}/status`가 `_threat_cache`를 노출
   (`threat_detected`, `threat_reason`, `screenshot` 등). 5분 후 캐시 자동
   정리.
5. **프론트엔드** `sandbox_browse_screen.dart`가 시작 직후부터 **2초 간격**
   으로 이 엔드포인트를 폴링(`_pollStatus`/`_startStatusPolling`,
   라인 69-102)하고, `threat_detected=true`면 `_buildThreatOverlay()`로
   차단 화면(스크린샷 + "이동한 주소가 알려진 위험 사이트로 확인되어
   세션을 자동으로 차단했어요" 안내)을 띄운다.

## Semaphore 동시성 — 알려진 미해결 이슈

`routers/sandbox.py`:
```python
if _AUTO_SEM._value == 0:          # 비공개 API, 사전 검사
    raise HTTPException(503, ...)
async with _AUTO_SEM:              # 검사~획득 사이 경합 가능
```
두 코루틴이 동시에 "마지막 슬롯이 비어있음"을 보고 둘 다 `async with`로
진입을 시도하면, asyncio.Semaphore의 설계상 한쪽은 즉시 503이 아니라 대기
큐에 들어간다(503 즉시 거부 의도와 미세하게 어긋남). 근본 해결은
`asyncio.Semaphore`가 "non-blocking try-acquire"를 공식 지원하지 않아서
구조적으로 어렵다 — 졸업작품 범위상 보류 상태(기능적 영향은 미미: 최악의
경우 동시 요청자가 503 대신 짧게 대기).
