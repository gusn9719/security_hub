# 시스템 구성도 쉽게 설명하기 (최종 발표용)

> 발표에서 시스템 구성도 이미지를 보여주면서 바로 말할 수 있게 정리한 버전입니다.
> 핵심은 **화면(UI) → 분석 기능 → DB → 다시 분석에 반영되는 피드백 구조**입니다.

## 1. 한 문장으로 말하면

저희 시스템은 문자 안에 들어 있는 URL을 뽑아내서, 그 URL이 안전한지 위험한지 여러 단계로 분석하고, 애매한 경우에는 샌드박스에서 직접 또는 AI가 대신 확인하게 해주는 피싱 URL 분석 시스템입니다.

그리고 분석 결과와 사용자 투표 결과가 DB에 쌓여서, 다음 분석 때 다시 반영됩니다. 그래서 단순히 한 번 검사하고 끝나는 구조가 아니라, 사용자 피드백으로 계속 보완되는 구조입니다.

---

## 2. 구성도 전체 흐름

구성도는 크게 3단으로 보면 됩니다.

```text
[화면 UI]
사용자가 문자를 넣고, 결과를 보고, 샌드박스나 투표 기능을 사용하는 부분

        ↓

[분석 기능]
문자에서 URL을 뽑고, 블랙리스트/화이트리스트/도메인 평판/점수화로 판단하는 부분

        ↓

[DB]
블랙리스트, 화이트리스트, 분석 결과, 샌드박스 결과, 투표, 회원 정보가 저장되는 부분

        ↺

DB에 쌓인 투표와 분석 결과가 다시 다음 분석에 반영됨
```

발표에서는 이렇게 말하면 됩니다.

> 위쪽은 사용자가 보는 화면이고, 가운데는 실제 분석이 일어나는 핵심 기능입니다.
> 아래쪽은 분석에 필요한 데이터와 분석 결과가 저장되는 DB입니다.
> 중요한 점은 DB에 저장된 결과가 끝이 아니라, 다음 분석에 다시 들어가서 판단을 더 정확하게 만든다는 점입니다.

---

## 3. 화면 UI 부분

화면 UI는 사용자가 시스템을 사용하는 입구입니다.

| 구성도 항목 | 쉽게 말하면 |
|---|---|
| 홈스크린 UI | 앱을 처음 켰을 때 보이는 메인 화면입니다. |
| 분석 결과 UI | 분석 결과가 안전, 의심, 위험 중 무엇인지 보여주는 화면입니다. 왜 그렇게 판단했는지도 카드로 보여줍니다. |
| 액션 UI | 위험하거나 의심스러운 결과가 나왔을 때 신고, 샌드박스 확인, 투표 같은 다음 행동을 선택하는 화면입니다. |
| 샌드박스 UI | 의심스러운 사이트를 격리된 가상 브라우저에서 직접 열어보거나 AI 테스트 결과를 확인하는 화면입니다. |
| 로그인 UI | 카카오 로그인 또는 비회원 사용을 선택하는 화면입니다. |

문자를 넣는 방법은 여러 가지입니다.

| 입력 기능 | 쉽게 말하면 |
|---|---|
| 공유하기 | 다른 앱에서 문자 내용을 공유해서 바로 분석합니다. |
| 클립보드 불러오기 | 복사한 문자 내용을 앱에서 감지해서 분석할 수 있게 합니다. |
| 문자 직접 입력 | 사용자가 문자 내용을 직접 입력합니다. |
| 문자 불러오기 | 문자함에서 최근 문자를 불러와 분석합니다. |
| 문자 알림 읽어오기 | 문자가 오면 알림을 통해 바로 분석할 수 있게 합니다. |

정리하면, 사용자는 문자를 직접 입력해도 되고, 복사하거나 공유하거나 문자함에서 불러와도 됩니다. 들어오는 방식만 다를 뿐, 결국 모두 같은 분석 파이프라인으로 들어갑니다.

---

## 4. 핵심 분석 기능 부분

가운데 영역이 저희 시스템의 메인 기능입니다. 발표에서는 이 순서대로 설명하면 가장 자연스럽습니다.

### 4-1. 로그인 / 회원가입

사용자는 카카오 로그인으로 회원가입하거나, 비회원으로도 사용할 수 있습니다.

로그인이 중요한 이유는 투표 기능 때문입니다. 익명 사용자는 기기 기준으로 제한하고, 카카오 로그인 사용자는 본인 인증을 거친 사용자이기 때문에 투표 신뢰도를 더 높게 볼 수 있습니다. 즉, 한 사람이 여러 번 투표해서 결과를 흔드는 것을 막기 위한 장치입니다.

### 4-2. 문자 전처리

문자가 들어오면 가장 먼저 문자 전체를 분석하는 것이 아니라, 그 안에서 **URL만 뽑아냅니다.**

예를 들어 문자 내용이 이렇게 들어오면:

```text
[택배 안내] 배송 주소를 확인해주세요. http://example.com/check
```

시스템은 문장 전체가 아니라 `http://example.com/check` 이 URL을 추출해서 분석합니다.

또 `bit.ly`, `forms.gle`, `t.co` 같은 단축 URL이면 실제 목적지 주소가 무엇인지 풀어서 확인합니다.

### 4-3. 블랙리스트 필터링

그다음 추출된 URL을 블랙리스트 DB와 비교합니다.

블랙리스트는 이미 피싱이나 악성으로 신고된 URL 목록입니다. 예를 들어 KISA C-TAS 같은 외부 위협 정보에서 받은 피싱 URL이 여기에 들어갑니다.

여기서 일치하면 바로 **위험**으로 판단합니다.

발표용으로는 이렇게 말하면 됩니다.

> 먼저 이미 알려진 피싱 URL인지 확인합니다. 이미 신고된 주소라면 더 복잡하게 볼 필요 없이 위험으로 판단합니다.

### 4-4. 화이트리스트 필터링

다음은 화이트리스트 DB와 비교합니다.

화이트리스트는 네이버, 카카오, 정부기관, 은행처럼 안전하다고 검증된 도메인 목록입니다. 이 목록에 정확히 맞거나, 허용된 하위 도메인 규칙에 맞으면 안전한 사이트로 볼 수 있습니다.

여기서 중요한 설계 원칙이 있습니다.

> 저희 시스템은 "모르면 안전"이라고 하지 않습니다.
> 확실히 검증된 화이트리스트에 있을 때만 안전이라고 합니다.
> 모르는 주소는 기본적으로 의심으로 보고, 추가 분석을 거칩니다.

이렇게 한 이유는 위험한 사이트를 안전하다고 잘못 판단하는 것이 가장 치명적이기 때문입니다.

### 4-5. 도메인 평판 분석

화이트리스트에도 없고 블랙리스트에도 없다면, 그 URL의 도메인 평판을 분석합니다.

여기서는 대표적으로 이런 정보를 봅니다.

| 분석 항목 | 의미 |
|---|---|
| 도메인 생성일 | 사이트가 만들어진 지 얼마 안 됐는지 확인합니다. 피싱 사이트는 짧게 만들고 버리는 경우가 많습니다. |
| SSL 인증서 발급일 | 보안 인증서가 너무 최근에 발급됐는지 확인합니다. |
| WHOIS 정보 | 도메인 등록 정보가 정상적으로 조회되는지 확인합니다. |

예를 들어 도메인이 생성된 지 30일도 안 됐고, SSL 인증서도 최근에 발급됐다면 “막 만든 피싱용 임시 사이트일 수 있다”고 보고 의심 신호로 반영합니다.

### 4-6. 휴리스틱 스코어링

그다음은 여러 위험 신호를 점수로 합산합니다. 이 부분이 휴리스틱 스코어링입니다.

현재 시스템은 위험 신호 25가지를 봅니다. 예시는 다음과 같습니다.

| 위험 신호 | 쉽게 말하면 |
|---|---|
| IP 주소 직접 접속 | `http://1.2.3.4`처럼 도메인 이름 없이 IP로 바로 접속시키는지 봅니다. |
| 악성 파일 확장자 | `.apk`, `.exe` 같은 실행 파일 다운로드를 유도하는지 봅니다. |
| 공식 사이트 흉내 | `naverr.com`처럼 유명 사이트 이름을 한두 글자 바꿔서 속이는지 봅니다. |
| 서브도메인 위장 | `naver.com.evil.kr`처럼 진짜 사이트처럼 보이게 주소를 꾸몄는지 봅니다. |
| 외국 문자 위장 | 한글이나 영문처럼 보이지만 실제로는 다른 문자로 만든 도메인인지 봅니다. |
| 새 도메인 | 만든 지 얼마 안 된 도메인인지 봅니다. |
| 과거 사용자 투표 | 다른 사용자들이 이 URL을 위험하다고 투표했는지 봅니다. |
| 샌드박스 위험 결과 | AI 자동 테스트나 샌드박스에서 위험 행동이 발견됐는지 봅니다. |

판정 기준은 간단히 이렇게 설명하면 됩니다.

```text
70점 이상  → 위험
30점 이상  → 의심
그 이하    → 화이트리스트가 아니면 의심
```

여기서도 핵심은 “점수가 낮다고 바로 안전이라고 하지 않는다”는 점입니다. 안전은 화이트리스트로 검증된 경우에만 줍니다.

### 4-7. 의심 사유 생성

분석 결과만 보여주면 사용자가 왜 위험한지 알기 어렵습니다. 그래서 시스템은 결과와 함께 이유 카드를 보여줍니다.

예를 들면:

- 최근에 만들어진 사이트입니다.
- 공식 사이트와 비슷한 이름으로 위장했습니다.
- 알려진 피싱 URL 목록에 포함되어 있습니다.
- 위험한 파일 다운로드가 의심됩니다.

이 설명은 AI가 즉석에서 지어내는 방식이 아니라, 미리 검증된 설명 문구 사전에서 선택해서 보여주는 방식입니다. 그래서 잘못된 설명이 만들어지는 위험을 줄였습니다.

### 4-8. 샌드박스 직접 탐방

분석 결과가 애매해서 의심으로 나온 경우, 사용자가 샌드박스에서 직접 사이트를 열어볼 수 있습니다.

샌드박스는 실제 스마트폰이나 개인 브라우저가 아니라, 서버 안의 격리된 가상 브라우저입니다. 그래서 사용자가 링크를 눌러보더라도 실제 기기에는 영향을 주지 않습니다.

발표에서는 이렇게 말하면 됩니다.

> 의심스러운 사이트를 실제 폰에서 누르는 것은 위험하기 때문에, 저희는 격리된 가상 브라우저 안에서 대신 열어볼 수 있게 했습니다.

### 4-9. 직접 탐방 중 위협 자동차단

샌드박스에서 사용자가 직접 탐방하는 동안에도 시스템이 실시간으로 감시합니다.

예를 들어:

- 알려진 피싱 사이트로 이동하는 경우
- 악성 파일 다운로드가 시작되는 경우
- 위험한 리다이렉트가 발생하는 경우

이런 상황이 감지되면 세션을 즉시 차단하고 사용자에게 경고합니다.

즉, 샌드박스는 단순히 “안전한 브라우저”만 제공하는 것이 아니라, 탐방 중 위험 행동도 자동으로 막습니다.

### 4-10. 사용자 투표

사용자는 샌드박스에서 확인한 뒤 이 사이트가 안전한지, 위험한지 투표할 수 있습니다.

이 투표 결과는 `샌드박스 투표 DB`에 저장되고, 다음에 같은 도메인이나 URL이 분석될 때 휴리스틱 점수에 반영됩니다.

이 부분이 저희 시스템의 차별점입니다.

> 정부나 외부 블랙리스트에 아직 등록되지 않은 신규 피싱 URL도, 사용자가 먼저 발견하고 위험하다고 투표하면 다음 분석부터 점수에 반영됩니다.

단, 투표 조작을 막기 위해 기기당 같은 도메인에는 한 번만 투표할 수 있고, 투표 하나만으로 바로 위험 확정이 되지는 않도록 점수 상한도 둡니다.

### 4-11. 샌드박스 AI 자동 테스트

사용자가 직접 탐방하지 않아도, AI 자동 테스트 기능을 사용할 수 있습니다.

이 기능은 가상의 이름, 전화번호, 카드번호 같은 테스트용 정보를 사용해서 사이트가 개인정보 입력을 유도하는지, 외부 폼으로 전송하는지, 자동 다운로드를 시도하는지 등을 확인합니다.

결과는 샌드박스 분석결과 DB에 저장되고, 필요하면 분석 점수에도 반영됩니다.

## 5. DB 부분

아래 영역은 시스템이 분석에 사용하는 데이터와 분석 결과를 저장하는 부분입니다.

| 구성도 항목 | 쉽게 말하면 |
|---|---|
| 블랙리스트 DB | 이미 신고된 피싱 URL 목록입니다. |
| 화이트리스트 DB | 안전하다고 검증된 도메인 목록입니다. |
| 평판분석 결과 캐시 DB | 도메인 생성일, SSL 인증서 같은 조회 결과를 일정 기간 저장합니다. 매번 다시 조회하지 않아도 되게 하기 위한 캐시입니다. |
| 분석 결과 DB | 사용자가 분석한 문자와 URL 결과가 저장되는 곳입니다. |
| 샌드박스 투표 DB | 사용자가 샌드박스에서 안전/위험으로 투표한 결과가 저장되는 곳입니다. |
| 샌드박스 분석결과 DB | AI 자동 테스트나 샌드박스 분석 결과가 저장되는 곳입니다. |
| 회원 DB | 카카오 로그인 사용자 정보가 저장되는 곳입니다. |

DB에서 가장 중요한 부분은 “저장하고 끝”이 아니라는 점입니다.

특히 샌드박스 투표 DB와 샌드박스 분석결과 DB는 다시 분석 로직으로 들어갑니다. 그래서 다음에 같은 URL이나 비슷한 도메인이 들어오면 과거 사용자 투표와 샌드박스 결과까지 반영해서 더 정확하게 판단합니다.

## 6. 발표 대본: 구성도 설명 버전

아래는 구성도 이미지를 띄워놓고 그대로 읽어도 되는 버전입니다.

> 저희 시스템 구성도입니다.
> 크게 위에서부터 화면 UI, 핵심 분석 기능, 데이터베이스 이렇게 3단 구조로 되어 있습니다.
>
> 먼저 사용자는 홈 화면에서 문자를 직접 입력하거나, 공유하기, 클립보드, 문자함 불러오기, 문자 알림 읽어오기 같은 방식으로 문자를 시스템에 넣을 수 있습니다.
>
> 메인 기능은 문자 분석입니다.
> 문자가 들어오면 먼저 문자 전체를 보는 것이 아니라, 그 안에서 URL만 추출합니다.
> 그리고 단축 URL이면 실제 목적지 주소까지 풀어서 확인합니다.
>
> 그다음 URL을 블랙리스트 필터링으로 먼저 검사합니다.
> 이미 신고된 피싱 URL 목록에 있으면 바로 위험으로 판단합니다.
> 반대로 화이트리스트 필터링에서는 네이버, 카카오, 정부기관처럼 확실히 검증된 안전 도메인인지 확인합니다.
> 저희 시스템은 모르는 주소를 바로 안전하다고 하지 않고, 화이트리스트에 있을 때만 안전으로 봅니다.
>
> 블랙리스트와 화이트리스트에 모두 없으면 도메인 평판 분석을 합니다.
> 이 사이트가 언제 만들어졌는지, SSL 인증서가 최근에 발급됐는지 같은 정보를 확인해서 새로 만들어진 피싱 사이트일 가능성을 봅니다.
>
> 그다음에는 휴리스틱 스코어링을 합니다.
> IP 주소로 직접 접속시키는지, 유명 사이트 이름을 비슷하게 흉내냈는지, 위험한 파일 다운로드를 유도하는지, 과거 사용자들이 위험하다고 투표했는지 같은 위험 신호 25가지를 점수로 합산합니다.
> 점수가 높으면 위험, 중간이면 의심으로 판단하고, 왜 그렇게 판단했는지도 의심 사유 카드로 보여줍니다.
>
> 만약 결과가 애매한 의심 단계라면 샌드박스를 사용할 수 있습니다.
> 사용자가 격리된 가상 브라우저에서 직접 사이트를 열어볼 수도 있고, AI가 가짜 정보를 넣어보면서 자동으로 테스트할 수도 있습니다.
> 탐방 중에 알려진 피싱 사이트로 이동하거나 악성 파일 다운로드가 감지되면 시스템이 자동으로 차단합니다.
>
> 마지막으로 이렇게 나온 분석 결과, 샌드박스 결과, 사용자 투표 결과는 각각의 DB에 저장됩니다.
> 특히 사용자 투표와 샌드박스 분석 결과는 다음 분석 때 다시 점수에 반영됩니다.
> 그래서 저희 시스템은 단순히 URL을 한 번 검사하는 도구가 아니라, 사용자 피드백과 샌드박스 결과를 계속 쌓아서 판단을 개선하는 구조입니다.

## 7. 아주 짧은 발표 버전

시간이 부족하면 아래처럼 줄여 말하면 됩니다.

> 저희 시스템은 문자 안에서 URL만 추출한 뒤, 그 URL을 블랙리스트, 화이트리스트, 도메인 평판, 휴리스틱 점수화 순서로 분석합니다.
> 블랙리스트에 있으면 위험, 화이트리스트에 있으면 안전으로 보고, 그 외에는 도메인 생성일이나 SSL 인증서, 주소 위장 여부 같은 위험 신호 25가지를 점수화해서 의심 또는 위험으로 판단합니다.
> 분석 결과는 DB에 저장되고, 의심스러운 URL은 샌드박스에서 직접 확인하거나 AI가 자동 테스트할 수 있습니다.
> 사용자가 샌드박스에서 안전/위험 투표를 하면 그 결과도 DB에 쌓이고, 다음 분석 때 다시 반영됩니다.
> 즉, 저희 시스템은 단순 검사 도구가 아니라 사용자 피드백으로 계속 보완되는 피싱 URL 분석 시스템입니다.

## 8. 예상 질문 답변

### Q1. 안전 판정은 어떻게 하나요?

화이트리스트에 있는 검증된 도메인일 때만 안전으로 판단합니다. 점수가 낮다고 무조건 안전이라고 하지 않습니다. 모르는 주소는 의심으로 보는 방식이라, 위험한 사이트를 안전하다고 잘못 판단하는 문제를 줄였습니다.

### Q2. 블랙리스트에 없는 신규 피싱 사이트는 어떻게 잡나요?

도메인 평판 분석과 휴리스틱 스코어링으로 잡습니다. 예를 들어 새로 만들어진 도메인인지, 유명 사이트를 흉내 낸 주소인지, IP 주소로 직접 접속시키는지, 위험한 파일을 내려받게 하는지 같은 신호를 합산합니다. 또 사용자가 샌드박스에서 위험하다고 투표하면 다음 분석부터 그 결과도 반영됩니다.

### Q3. 왜 샌드박스가 필요한가요?

URL 분석만으로는 애매한 경우가 있습니다. 이때 실제 사용자 기기에서 누르는 것은 위험하기 때문에, 격리된 가상 브라우저에서 대신 열어보게 합니다. 직접 확인 중에도 위험 행동이 감지되면 자동으로 차단합니다.

### Q4. AI 자동 테스트는 무엇을 하나요?

사람 대신 가짜 이름, 전화번호, 카드번호 같은 테스트 정보를 입력해보면서 개인정보 탈취 폼, 외부 전송, 자동 다운로드 같은 위험 행동이 있는지 확인합니다.

### Q5. 투표 조작은 어떻게 막나요?

기기당 같은 도메인에는 한 번만 투표할 수 있게 하고, 다수 의견이 명확할 때만 점수에 반영합니다. 또 투표 하나만으로 바로 위험 확정이 되지 않도록 점수 상한을 두었습니다. 카카오 로그인 사용자는 본인 인증을 거친 사용자라 익명보다 더 높은 신뢰도로 반영합니다.

## 9. 발표에서 꼭 강조할 포인트

1. 문자 전체가 아니라 **URL만 추출해서 분석**한다.
2. 분석 순서는 **블랙리스트 → 화이트리스트 → 도메인 평판 → 휴리스틱 점수화**다.
3. 안전 판정은 신중하게 한다. **화이트리스트에 있을 때만 안전**이다.
4. 의심스러우면 **샌드박스에서 직접 확인하거나 AI 자동 테스트**를 한다.
5. 분석 결과와 투표 결과가 DB에 남고, **다음 분석에 다시 반영**된다.
6. 그래서 이 시스템은 단순 차단 도구가 아니라 **사용자 피드백으로 계속 개선되는 피싱 URL 분석 시스템**이다.
